# KMP 통신사 이탈 데이터 정의서

## 1. 데이터셋 개요

이 데이터셋은 한국미디어패널(KMP) 2020~2025 개인조사 원본을 이용해 만든 통신사 전환(transition) 기반 패널 데이터셋이다.

한 행은 한 개인(`pid`)의 전년도(`t-1`) 정보와 다음년도(`t`) 통신사 상태를 연결한 기록이다.

예를 들어,
- `year_t0 = 2020`
- `year_t1 = 2021`

이면 이 행은 2020년의 개인 특성을 사용해서 2021년에 통신사가 바뀌었는지를 나타낸다.

즉, 이 데이터셋은 일반적인 “사람 1명 = 1행” 구조가 아니라,  
“사람 1명 = 연도 전환 구간별 여러 행” 구조다.

예를 들어 어떤 사람이 2020~2025에 모두 존재하면 아래 전환 구간이 생성될 수 있다.

- 2020 → 2021
- 2021 → 2022
- 2022 → 2023
- 2023 → 2024
- 2024 → 2025

따라서 한 사람이 최대 5행까지 등장할 수 있다.

---

## 2. 데이터 생성 기준

### 한 행의 의미

한 행은 `(pid, year_t0 → year_t1)` 전환을 의미한다.

### 입력 피처 사용 원칙

모든 설명변수는 반드시 `t-1` 시점 값만 사용했다.

즉:
- `year_t0` = 설명변수가 속한 연도
- `year_t1` = 라벨이 결정되는 다음 연도

이 구조는 미래 정보 누수(data leakage)를 막기 위한 것이다.

### 라벨 정의

- `churn_any`: 전년도와 다음년도 통신사가 다르면 1
- `churn_to_mvno`: 전년도 메이저 통신사(1,2,3) 사용자가 다음년도 MVNO(4)로 이동하면 1

---

## 3. 전처리 규칙

### 결측 처리

원본 코드북에서 자주 쓰이는 결측/무응답 코드는 전처리 시 결측으로 처리했다.

- `9999` → 결측
- `9998` → 결측
- `9997` → 결측

### 문자열 정리

- NBSP(`\xa0`) 제거
- 앞뒤 공백 제거

### 통신사 유효값 필터

통신사 변수는 아래 값만 유효값으로 사용했다.

- `1` = SKT
- `2` = KT
- `3` = LG U+
- `4` = 알뜰폰 서비스(MVNO)

### 이진 변수 변환

아래 변수들은 원래 코드북상 `1=가입`, `2=미가입` 구조였고, 모델링 편의를 위해 `1/0`으로 변환했다.

- `a03024_tminus1`
- `a03026_tminus1`
- `c02003_tminus1`

변환 기준은 다음과 같다.

- 원본 `1` → 전처리 후 `1`
- 원본 `2` → 전처리 후 `0`

### 중복 처리

같은 `pid`가 같은 전환 구간(`year_t0`, `year_t1`)에 중복되면 하나만 남겼다.

## 5. 컬럼별 상세 설명

## 5.1 `pid`

### 의미
개인 식별자

### 역할
같은 사람을 연도 간 연결하기 위한 기준 키다.

### 해석
한 사람이 여러 전환 구간에 반복 등장할 수 있으므로,  
이 데이터셋은 개별 독립 샘플의 단순 모음이 아니라 패널 구조를 가진다.

### 분석 시 주의
모델링 시 랜덤 분할이 아니라 `pid` 기준 그룹 분할을 해야 한다.  
같은 사람의 여러 행이 train/test에 동시에 들어가면 누수 가능성이 생긴다.

---

## 5.2 `year_t0`

### 의미
기준 시점 연도(`t-1`)

### 역할
설명변수들이 속한 연도다.

### 해석
예를 들어 `year_t0 = 2022`이면 `_tminus1` 변수들은 모두 2022년 값이다.

---

## 5.3 `year_t1`

### 의미
다음 시점 연도(`t`)

### 역할
이탈 여부가 판단되는 연도다.

### 해석
예를 들어 `year_t1 = 2023`이면,  
2022년 정보를 이용해 2023년에 통신사가 바뀌었는지를 보는 구조다.

---

## 5.6 `churn_any`

### 정의
전년도와 다음년도 통신사가 다르면 1, 같으면 0

### 계산 기준
- `telco_t0 != telco_t1` → `1`
- `telco_t0 == telco_t1` → `0`

### 의미
가장 넓은 의미의 통신사 이탈 여부다.

### 포함되는 사례
- 메이저 → 메이저 이동
- 메이저 → MVNO 이동
- MVNO → 메이저 이동
- MVNO → 다른 통신사 이동

### 예시
- `1 → 1` : `0`
- `2 → 1` : `1`
- `3 → 4` : `1`
- `4 → 2` : `1`

### 분석 관점
통신사 변경 자체를 예측하는 기본 타깃이다.  
`churn_to_mvno`보다 양성 비율이 높아 baseline 모델을 만들기 쉽다.

---

## 5.7 `churn_to_mvno`

### 정의
전년도 메이저 통신사(1,2,3)를 쓰다가 다음년도에 MVNO(4)로 이동하면 1

### 계산 기준
- `telco_t0 in [1, 2, 3]` and `telco_t1 == 4` → `1`
- 그 외 → `0`

### 의미
전체 이탈 중에서도 메이저 통신사에서 알뜰폰으로 이동한 경우만 따로 분리한 타깃이다.

### 예시
- `1 → 4` : `1`
- `2 → 4` : `1`
- `3 → 4` : `1`
- `2 → 3` : `0`
- `4 → 1` : `0`

### 분석 관점
가격 절감형 이동이나 요금 민감형 이탈을 더 직접적으로 볼 수 있는 타깃이다.

### 주의
양성 비율이 낮아 클래스 불균형이 심할 가능성이 높다.  
평가 시 Accuracy보다 PR-AUC, Recall, F1 등을 함께 보는 것이 좋다.

---

## 5.8 `a03002_tminus1`

### 원본 변수
`pYYa03002`

### 코드북 변수 설명
스마트폰 구분 - 스마트폰 1

### 의미
`t-1` 시점 사용 중인 스마트폰의 통신 기술 세대/유형

### 코드값 의미
- `1` = 3G 스마트폰
- `2` = LTE 스마트폰
- `3` = LTE-A 스마트폰
- `4` = 5G 스마트폰
- `9999` = 모름/무응답

### 전처리 후 의미
결측 코드 `9999`는 제거했다.

### 무엇을 나타내는가
사용자가 어떤 세대의 모바일 기술 환경을 사용하고 있는지를 나타낸다.

### 해석 포인트
이 변수는 단순 기기명이 아니라,
- 통신 기술 수준
- 최신 기기 채택 여부
- 고가 요금제 사용 가능성
과 연결될 수 있다.

### 분석 관점
5G 사용자는 상대적으로 최신 단말, 고가 요금제, 높은 데이터 사용 패턴과 연결될 가능성이 있다.  
반대로 구형 기술 사용자와 최신 기술 사용자는 이탈 원인이 다를 수 있다.

### 주의
이 값은 연속형 숫자가 아니라 범주형 코드로 보는 것이 적절하다.

### 주의

## 5.13 `c02003_tminus1`

### 원본 변수
`pYYc02003`

### 코드북 변수 설명
휴대폰 결합상품 가입 여부

### 원본 코드값 의미
- `1` = 가입
- `2` = 미가입
- `9999` = 모름/무응답

### 전처리 후 코드값 의미
- `1` = 가입
- `0` = 미가입

### 무엇을 나타내는가
휴대폰이 다른 통신 상품과 결합되어 있는지를 나타낸다.

### 해석 포인트
결합상품은 보통 인터넷, IPTV 등과 함께 묶여 할인되는 경우가 많기 때문에 이탈 장벽과 관련될 수 있다.

### 분석 관점
결합상품 가입자는 락인(lock-in) 효과 때문에 통신사 변경 가능성이 낮을 수 있다.  
특히 메이저 통신사에서 MVNO로 이동하는 경우 강한 제약 요인이 될 수 있다.

### 주의
이 변수도 원본 `1/2`가 전처리 후 `1/0`으로 바뀌었다.

## 5.14 `c02001_tminus1`

### 원본 변수
`pYYc02001`

### 코드북 변수 설명
휴대폰 요금 부담자

### 코드값 의미
- `1` = 본인(소득 및 용돈에서) 직접 부담
- `2` = 회사가 전액 부담
- `3` = 회사가 일부 지원
- `4` = 가족이나 타인이 전액 부담
- `5` = 가족이나 타인이 일부 지원
- `6` = 기타

### 무엇을 나타내는가
실제로 누가 휴대폰 비용을 부담하는지를 나타낸다.

### 해석 포인트
가격 민감도는 요금을 누가 내는지에 따라 크게 달라질 수 있다.

### 분석 관점
본인이 직접 부담하면 요금 인상이나 할인 혜택에 더 민감할 수 있다.  
반대로 회사 지원이나 가족 부담이 있는 경우 개인의 직접적인 가격 민감도가 낮을 수 있다.

### 주의
이 변수 역시 숫자의 크기보다 범주 자체의 의미가 중요하다.

---

## 6. 해석 포인트

### 1) 이 데이터는 현재 상태가 아니라 다음 해 변화 예측용 구조다

설명변수는 모두 이전 연도 값이고, 라벨은 다음 연도 통신사 변화로 정의된다.

### 2) `telco_t1`은 입력변수가 아니다

`telco_t1`은 다음 해 결과 자체이므로 모델 입력에 사용하면 안 된다.

### 3) `churn_any`와 `churn_to_mvno`는 목적이 다르다

- `churn_any`: 모든 통신사 이동
- `churn_to_mvno`: 메이저 → 알뜰폰 이동만 따로 추출

즉 두 라벨은 같은 듯 보이지만 분석 목적이 다르다.

### 4) 숫자처럼 보여도 실제로는 코드형 변수인 컬럼이 있다

특히 아래 변수들은 연속형 수치가 아니라 범주형 또는 순서형 코드로 해석하는 편이 적절하다.

- `a03002_tminus1`
- `c01002_tminus1`
- `c01004_tminus1`
- `c02001_tminus1`

### 5) 사람 단위 패널 구조를 고려해야 한다

같은 `pid`가 여러 행에 등장하므로, 단순 IID 데이터처럼 다루면 평가가 왜곡될 수 있다.

## 8. 요약

이 데이터셋은 KMP 2020~2025 원본에서 개인별 연도 전환을 기준으로 만든 통신사 이탈 예측용 패널 데이터다.

핵심 특징은 다음과 같다.

- 한 행은 한 개인의 `t-1 → t` 전환 정보다.
- 설명변수는 모두 `t-1` 값만 사용했다.
- `churn_any`와 `churn_to_mvno` 두 가지 라벨을 만든다.
- 일부 변수는 코드북의 `1/2` 값을 전처리 후 `1/0`으로 바꿨다.
- 여러 변수는 숫자처럼 보여도 실제로는 범주형/순서형 코드다.
- 같은 사람이 여러 행에 등장하므로 `pid` 기준 분할이 중요하다.
